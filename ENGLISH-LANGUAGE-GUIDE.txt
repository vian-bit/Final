📝 TRANSLATION REFERENCE:
==========================

Use lang/en/messages.php for translations:

In Blade Templates:
-------------------
Instead of: "Sistem Absensi Hotel"
Use: {{ __('messages.app_name') }}

Instead of: "Dashboard"
Use: {{ __('messages.dashboard') }}

Instead of: "Simpan"
Use: {{ __('messages.save') }}


In Controllers:
---------------
Instead of: 'Data berhasil disimpan'
Use: __('messages.data_saved')

Instead of: 'Login berhasil'
Use: __('messages.login_success')


Example Usage:
--------------
// In Blade
<h1>{{ __('messages.app_name') }}</h1>
<button>{{ __('messages.save') }}</button>

// In Controller
return redirect()->back()->with('success', __('messages.data_saved'));

💡 BEST PRACTICES:
==================

1. Always use language files:
   - Don't hardcode text in views
   - Use {{ __('messages.key') }}
   - Easier to maintain

2. Consistent terminology:
   - Use LANGUAGE-MAPPING.txt as reference
   - Same term = same translation
   - Example: Always use "Attendance" for "Absensi"

3. Context matters:
   - "Simpan" can be "Save" or "Submit"
   - Choose based on context
   - Be consistent

4. Test thoroughly:
   - Check all pages
   - Test all features
   - Verify messages
