❌ ERROR YANG TERJADI:
======================

419 | PAGE EXPIRED

Error ini muncul saat login atau submit form.


🔍 PENYEBAB:
============

1. CSRF Token expired (token keamanan Laravel)
2. Session expired atau tidak valid
3. Cache perlu dibersihkan
4. Server di-restart tapi browser masih pakai token lama
5. Cookie browser bermasalah


✅ SOLUSI CEPAT (OTOMATIS):
===========================

Double-click: FIX-419-ERROR.bat

Script akan:
1. Clear semua cache
2. Recreate session table
3. Reset CSRF token
4. Siap digunakan


✅ SOLUSI MANUAL:
=================

LANGKAH 1 - Clear Cache:
-------------------------
Buka terminal di folder project, jalankan:

php artisan cache:clear
php artisan config:clear
php artisan view:clear
php artisan route:clear


LANGKAH 2 - Restart Server:
----------------------------
1. Stop server (Ctrl+C)
2. Jalankan lagi: php artisan serve


LANGKAH 3 - Clear Browser:
---------------------------
1. Buka browser baru atau Incognito/Private mode
2. Atau clear cookies: Ctrl+Shift+Delete
3. Pilih: Cookies dan Cache
4. Clear


LANGKAH 4 - Login Ulang:
-------------------------
1. Akses: http://localhost:8000
2. Login dengan akun demo
3. Seharusnya berhasil

💡 TIPS MENCEGAH ERROR 419:
============================

1. Jangan gunakan tombol Back di browser setelah login
2. Jangan biarkan halaman login terbuka terlalu lama
3. Setelah restart server, refresh browser (F5)
4. Gunakan Incognito mode untuk testing
5. Clear cache browser secara berkala

🔧 TROUBLESHOOTING LANJUTAN:
=============================

Masih Error 419?
----------------

SOLUSI A - Hard Refresh Browser:
1. Tekan Ctrl+Shift+R (hard refresh)
2. Atau Ctrl+F5
3. Login ulang


SOLUSI B - Clear Browser Data:
1. Tekan Ctrl+Shift+Delete
2. Pilih "All time"
3. Centang: Cookies, Cache
4. Clear data
5. Restart browser
6. Login ulang


SOLUSI C - Gunakan Browser Lain:
1. Coba Chrome jika pakai Firefox
2. Atau sebaliknya
3. Atau gunakan Incognito/Private mode


SOLUSI D - Recreate Session Table:
1. Buka terminal
2. Jalankan:
   php artisan session:table
   php artisan migrate
3. Restart server
4. Login ulang


SOLUSI E - Check .env:
1. Pastikan APP_KEY terisi
2. Jika kosong, jalankan:
   php artisan key:generate
3. Restart server

❓ FAQ:
=======

Q: Kenapa error 419 muncul?
A: CSRF token expired. Laravel menggunakan token untuk keamanan.

Q: Apakah data saya hilang?
A: Tidak, data aman. Hanya perlu login ulang.

Q: Berapa lama session berlaku?
A: Default 120 menit (2 jam). Bisa diubah di .env.

Q: Apakah harus clear cache setiap kali?
A: Tidak. Hanya saat error atau setelah update code.

Q: Bagaimana cara perpanjang session?
A: Edit .env, ubah SESSION_LIFETIME=120 ke nilai lebih besar.

🔐 KEAMANAN:
============

Error 419 adalah fitur keamanan Laravel untuk:
- Mencegah CSRF attack
- Melindungi form dari spam
- Memastikan request valid

Jangan disable CSRF protection!

📞 JIKA MASIH BERMASALAH:
=========================

1. Cek log error:
   storage/logs/laravel.log

2. Cek APP_KEY di .env (harus terisi)

3. Cek SESSION_DRIVER di .env (harus: database)

4. Pastikan tabel sessions ada di database

5. Restart Laragon (Stop All > Start All)

6. Jalankan: FIX-419-ERROR.bat
